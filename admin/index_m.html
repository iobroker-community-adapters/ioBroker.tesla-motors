Fleet Telemetry (MQTT Bridge)

Configure a local vehicle-command proxy plus an MQTT-fed Fleet Telemetry server. The adapter will then update the existing Tesla states from MQTT instead of using regular vehicle_data polling.

The Fleet Telemetry field selection is configured on the Fleet Telemetry fields tab. The adapter stores that selection internally as JSON for compatibility with older versions.

Fleet Telemetry fields

Select exactly which Fleet Telemetry fields should be configured on the vehicle and define the minimum update interval per field. Tesla still sends values only when the value changed and the configured interval has elapsed.

Fields tagged with default are the adapter preset. Fields tagged with vehicle_location require the vehicle_location OAuth scope. Unmapped fields are stored under <VIN>.telemetry.fields.<FieldName> as raw telemetry values.
